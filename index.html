Project Overview

The Online Library Management System is a desktop-based application that allows users to browse, search, borrow, and return media items such as books, films, and magazines. The system tracks availability, borrower information, and due dates while providing clear visual feedback through the user interface.

The project focuses on applying advanced programming concepts such as modular design, client–server architecture, and separation of concerns.

Technologies Used

  • Python – Core programming language
  • Flask – RESTful backend API
  • PyQt5 – Desktop graphical user interface
  • JSON – Lightweight persistent data storage
  • Git & GitHub – Version control
  • GitHub Pages – Static documentation website

Key Features

  • Browse media items by category
  • Search media using partial name matching
  • Borrow and return functionality
  • Borrower name and due date tracking
  • Automatic overdue detection
  • Visual indicators for borrowed and overdue items
  • Media cover image support
  • Dark mode in the desktop application
  • Client-side dark mode for the documentation website

System Architecture

The system follows a three-layer architecture to ensure maintainability and separation of concerns:

  • Frontend (PyQt5): Handles user interaction and presentation logic.
  • Backend (Flask REST API): Implements business logic and exposes RESTful endpoints.
  • Data Layer (JSON): Stores media and borrowing data in a lightweight format.
